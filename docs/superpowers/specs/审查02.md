## 四、架构问题 🟡

### 4.1 层间依赖方向存疑

**问题：** 策略层依赖分析结果，但分析结果中包含评分逻辑，职责边界模糊。

**当前设计：**
```
数据层 → 业务层 → 策略层 → 交易层
```

**潜在问题：**
- `AnalyzedOption.score` 是谁计算的？`analyzer` 还是 `evaluator`？
- 评分逻辑分散在两处，不利于维护

**建议明确职责边界：**

```python
# core/analyzer.py - 只负责数据清洗和基础指标
class OptionAnalyzer:
    def analyze(self, quotes: List[OptionQuote]) -> List[AnalyzedOption]:
        """
        职责：
        - 计算基础指标（ITM/OTM、杠杆率等）
        - 数据清洗和验证
        - 不涉及评分
        """
        pass

# strategy/evaluator.py - 负责评分和排序
class StrategyEvaluator:
    def evaluate(self, analyzed: List[AnalyzedOption],
                 market_context: MarketContext) -> List[ScoredOption]:
        """
        职责：
        - 多因子评分
        - 市场环境加权
        - 机会排序
        """
        pass
```
